Техническое задание (ТЗ) для мебельного сайта
1. Введение
Цель данного ТЗ — разработка функционального и визуально привлекательного веб-сайта для компании, занимающейся продажей мебели. Сайт должен обеспечивать удобный интерфейс для пользователей и эффективные инструменты для администрирования.

2. Общие требования
Сайт должен быть доступен на русском и английском языках.
Дизайн сайта должен соответствовать современным тенденциям в области веб-дизайна и быть адаптивным для мобильных устройств.
3. Функциональные требования
3.1. Главная страница
На главной странице должны отображаться:
Логотип компании.
Основное меню с разделами: «Каталог», «О нас», «Контакты», «Блог», «Корзина».
Слайдер с акциями и новинками (не менее трех изображений).
Блок с популярными товарами (не менее четырех позиций).
3.2. Каталог товаров
Каталог должен включать:
Фильтры по категориям (например, «Мебель для гостиной», «Кухонная мебель»).
Возможность сортировки по цене, популярности и новизне.
Страницы товара с подробным описанием, характеристиками, ценой и кнопкой «Добавить в корзину».
3.3. Корзина
Корзина должна включать:
Список добавленных товаров с возможностью изменения количества или удаления товара.
Итоговую стоимость заказа.
Кнопку для перехода к оформлению заказа.
3.4. Оформление заказа
Оформление заказа должно включать:
Форму для ввода контактной информации (имя, телефон, адрес доставки).
Выбор способа оплаты (наличный расчет, банковская карта).
Кнопку подтверждения заказа.
4. Нефункциональные требования
4.1. Производительность
Время загрузки страниц не должно превышать 3 секунд при стандартном интернет-соединении.
4.2. Безопасность
Все данные пользователей должны передаваться по защищенному протоколу HTTPS.
Необходимо реализовать защиту от SQL-инъекций и XSS-атак.
4.3. SEO-оптимизация
Все страницы должны быть оптимизированы для поисковых систем (метатеги, заголовки, alt-теги для изображений).
5. Проверяемость требований
Каждый пункт функциональных требований должен быть протестирован на соответствие следующим критериям:

Полнота: все заявленные функции реализованы.
Конкретность: каждая функция работает согласно описанию.
Непротиворечивость: отсутствуют конфликты между различными функциями.
Проверяемость: каждая функция может быть протестирована на работоспособность.
