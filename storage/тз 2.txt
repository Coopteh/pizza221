Раздел функциональных требований ТЗ для сайта мебельного магазина
1. Список функций системы
Отображение каталога мебели:

Система должна отображать список доступной мебели с возможностью фильтрации по категориям (например, диваны, столы, стулья).
Пользователь может просматривать детальную информацию о каждом товаре (фото, описание, цена, характеристики).
Поиск товаров:

Пользователь может вводить запрос в строку поиска для нахождения конкретных товаров.
Система должна отображать результаты поиска, соответствующие введенному запросу.
Регистрация нового пользователя:

Пользователь может зарегистрироваться, указав свои контактные данные.
Система должна отправлять подтверждение регистрации на электронную почту.
Авторизация пользователей:

Пользователь может войти в систему, используя логин и пароль.
Система должна проверять введенные данные и предоставлять доступ к личному кабинету.
Добавление товара в корзину:

Пользователь может добавлять выбранные товары в корзину.
Система должна обновлять количество товаров в корзине и отображать общую стоимость.
Оформление заказа:

Пользователь может перейти в корзину и нажать кнопку «Оформить заказ».
Система должна запрашивать адрес доставки, контактные данные и способ оплаты.
Оплата и подтверждение заказа:

Система должна интегрироваться с платёжными системами для обработки транзакций.
После успешной оплаты система должна генерировать чек и отправлять уведомление на электронную почту.
Интеграция с внешними сервисами доставки:

Система должна отправлять данные о заказе (адрес, контактный телефон) в службу доставки.
Подтверждение от службы доставки должно возвращаться в систему в реальном времени.
Управление личным кабинетом:

Пользователь может просматривать историю заказов и изменять свои контактные данные.
Система должна предоставлять возможность изменения пароля.
2. Описание сценариев использования
Сценарий оформления заказа:

Пользователь заходит на сайт и просматривает каталог мебели.
Выбирает интересующий товар и добавляет его в корзину.
Переходит в корзину и проверяет выбранные товары.
Нажимает кнопку «Оформить заказ».
Указывает адрес доставки и контактные данные.
Выбирает способ оплаты (онлайн или наложенный платеж).
Подтверждает заказ.
Система отображает подтверждение заказа и отправляет уведомление на электронную почту.
Сценарий регистрации нового пользователя:

Пользователь заходит на сайт и нажимает кнопку «Регистрация».
Заполняет форму с контактными данными (имя, электронная почта, пароль).
Нажимает кнопку «Зарегистрироваться».
Система отправляет письмо с подтверждением на указанную электронную почту.
Пользователь переходит по ссылке в письме для подтверждения регистрации.
3. Уточнение требований к взаимодействию с другими компонентами
Система оплаты:

Сайт должен интегрироваться с платёжными системами (например, Stripe, PayPal).
Передача данных о заказе и сумме должна происходить через API платёжной системы.
Система должна обрабатывать успешные и неуспешные транзакции и уведомлять пользователя о статусе оплаты.
Сервис доставки:

После оформления заказа система должна отправлять данные о заказе (адрес, контактный телефон) в систему логистики.
Подтверждение от службы доставки должно возвращаться в систему в реальном времени, чтобы пользователь мог отслеживать статус доставки.
База данных:

Система должна взаимодействовать с базой данных для хранения информации о пользователях, товарах, заказах и платежах.
Все операции с базой данных должны быть защищены от SQL-инъекций и других уязвимостей.
